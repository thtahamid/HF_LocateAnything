1. Overview

The core problem LocateAnything solves — and how it differs from standard VLM grounding.

The Bottleneck in Standard VLMs

Vision-Language Models traditionally formulate grounding as a next-token-prediction problem: a 2D bounding box (4 floats) is serialized into many 1D tokens (each digit/punctuation becomes a token), decoded one at a time. This creates two issues:

Token-by-Token (Standard)

Each coordinate digit predicted independently. Generations are strictly sequential — one token per model forward pass per box. The model has no internal notion that "512" and "384" are coupled coordinates.

Parallel Box Decoding (LocateAnything)

An entire 4-coordinate box is decoded as a single atomic unit in one parallel step. The model sees all 4 coordinates simultaneously and learns geometric dependencies between them.

6. On-Demand Inference Modes

Dynamic tradeoff between speed and precision via hybrid MTP+AR decoding.

is_valid_box_frame() checks p(<box>)≥0.6 & top-k shape → 3. If 'legal_box' → decode coordinates via decode_bbox_avg() (top-k weighted average) 4. If 'illegal_box' → discard, switch to AR mode for this block 5. If 'empty_box' → output <box>none</box> (object absent)

Bounding Box Decoding: Top-k Weighted Average

Instead of taking the argmax at each coordinate position, LocateAnything averages over the top-k candidate coordinate tokens weighted by their probability:

decoded_coord_i = Σₖ (prob_k × token_id_k) / Σₖ prob_k   for position ∈ {1,2,3,4}
final_coords = [<box>, coord_1, coord_2, coord_3, coord_4, </box>]

In hybrid mode, positions where the top-1 probability < 0.9 and the valid-token span exceeds 60 bins are detected as "abnormal" and defaulted to 0 (flagging for AR fallback logic in downstream).

7. Native Resolution Handling

Moon-ViT preserves fine-grained spatial details by operating at the image's original resolution.

Processing Pipeline

StepComponentDetail
1. Loadfetch_image()Loads image at native resolution. Handles local files, HTTP URLs, base64, LMDB stores. Converts to RGB.
2. TokenizeImage ProcessorCalls LocateAnythingImageProcessor with the image at native resolution. Computes image_grid_hws array (H×W in patch units).
3. EmbedMoon-ViTProduces visual token sequence at native grid resolution. Applies 2×2 token merge.
4. ProjectMLP ×21152 → 2048 dims, SiLU activation. Positional embeddings interpolated from 64×64 init to native grid.
5. DecodeQwen2.5Receives projected visual tokens + text query → emits sequence of 6-token blocks at block_size=6.
