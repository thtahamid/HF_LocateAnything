1. Project Overview

What are we testing, on what data, and why does it matter?

Goal / Objective

Establish a quantitative baseline for how well the LocateAnything-3B model — a general-purpose vision-language grounding system trained on natural images — performs on electron microscopy (EM) connectomics datasets. Compare directly with the microscopy-domain tool micro-sam to measure the domain gap and identify where fine-tuning / LoRA adaptation is most needed.

Setup the Model

Configure LocateAnything-3B (NVIDIA/HF) in hybrid decoding mode. Bf16, A100-40/80 GB, single-slice inference pipeline with native-resolution MoonViT.

Test on Lucchi++ / Kasthuri++

Run zero-shot inference per-2D-slice through both EM volumes. Record boxes, parse <box> tokens, project to pixel coordinates, compute metrics.

Report & Compare

Generate segmentation reports in /docs: per-slice mSA, F1@{IoU}, 3D adaptive Rand. Head-to-head comparison against micro-sam as SOTA microscopy baseline.

Parallel Box Decoding (PBD) — the core innovation. Instead of emitting coordinates token-by-token (e.g., <box> 1 23 45 6 </box>), the model treats each 6-token block as an atomic unit decoded in one parallel step, yielding a 2.5× throughput improvement over autoregressive methods.

4. Experiment Design

Five complementary experiments characterize every dimension of baseline performance.

Experiment Execution Pipeline

E1: Zero-Shot Baseline

Run LocateAnything-3B in hybrid mode on every test-slice. Record raw boxes. Compute F1 metrics. This is the reference point for everything else.

E2: Prompt Sensitivity

Test 6 prompt variants: noun form, descriptive, negative control, pointing mode. Measure F1 deltas to quantify how much language affects zero-shot EM accuracy.

E3: 3D Wrapping

Link per-slice boxes across Z with Hungarian matching on IoU overlap. Convert tracked boxes to mask via ellipsoid fill. Evaluate 3D mSA, Rand error, CREMI-score.

E4: micro-sam Comparison

Fair head-to-head. Same splits, same metrics. Evaluate both at box-level and mask-level. vit_b_em fine-tuned vs zero-shot LA.

E5: Resolution Scaling

MoonViT's native-resolution ViT should preserve spatial detail. Test native 1024×768 vs 2× / 4× downsampled vs tiled 512×512, measure F1 impact.

● micro-sam

  • TypeFoundation segmentation
  • OutputInstance masks
  • 3D supportSlice-by-slice propagation
  • PromptPoint / box / auto
  • DomainMicroscopy-tuned (em/lm)
  • VRAM~2-6 GB (vit_b → vit_h)
  • Speed~1-3 BPS (AIS)
  • Lucchi++Evaluated (mSA)
  • Kasthuri++Protocol defined
Critical Protocol Mismatch: LocateAnything outputs bounding boxes, not pixel-level instance segmentation masks. For fair comparison, micro-sam masks must be reduced to boxes (for F1@IoU), and LocateAnything boxes must be converted to pseudo-masks (ellipsoid fill or SAM-decoder bridge) for mSA / Rand error. Both transformations are lossy — document their contribution to inter-method variance.

7. System Resources & Code Blueprint

Hardware budget, memory profile, and the inference/evaluation script skeleton

GPU Memory Allocation (A100-40GB, Experiment 1)

# ── inference.py ── Run LocateAnything-3B on Lucchi++ slices ──

from PIL import Image
from transformers import AutoModel, AutoTokenizer
import torch, h5py, json, numpy as np
from elf.evaluation import mean_segmentation_accuracy, dice_score

# ① Load model (bf16, A100-40GB)
model = AutoModel.from_pretrained(
    "nvidia/LocateAnything-3B",
    torch_dtype=torch.bfloat16,
    trust_remote_code=True,
).cuda().eval()
tokenizer = AutoTokenizer.from_pretrained("nvidia/LocateAnything-3B")

# ② Load Lucchi++ test volume (one stack: 165 × 768 × 1024)
with h5py.File("/datasets/lucchi/test.h5") as f:
    volume = f["volume"][:]    # shape (165, 768, 1024)
    labels = f["labels"][:]    # binary mitochondria mask

# ③ Per-slice inference
prompt = "Locate all the instances that matches the following description: mitochondria."
results = {}
for z in range(volume.shape[0]):
    img = Image.fromarray(volume[z]).convert("RGB")
    inputs = processor(text=prompt, images=img, return_tensors="pt").cuda()
    with torch.no_grad():
        out = model.generate(**inputs, mode="hybrid", max_new_tokens=2048)
    boxes = parse_boxes(out[0].cpu().tolist(), img.width, img.height)
    metric = eval_slice(boxes, labels[z])
    results[z] = {*"boxes": boxes, **metric}

# ④ Report
f1_scores = [r["f1_50"] for r in results.values()]
print("Mean F1@0.5 = {np.mean(f1_scores):.3f} ± {np.std(f1_scores):.3f}")
json.dump(results, open("/docs/discussions/ml-intern/results_e1.json", "w"))
# ── metrics.py ── Box + Mask metrics for EM evaluation ──

def eval_slice(pred_boxes, gt_mask):
    """Compute F1@0.5, F1@0.75, and mask-level mSA per slice."""
    from skimage.measure import label, regionprops
    from scipy.optimize import linear_sum_assignment

    gt_inst = label(gt_mask)
    gt_boxes = [r.bbox for r in regionprops(gt_inst)]  # (min_r,min_c,max_r,max_c)
    iou_mat = compute_iou_matrix(gt_boxes, pred_boxes)
    row, col = linear_sum_assignment(-iou_mat)

    tp50 = sum(iou_mat[r, c] >= 0.5 for r, c in zip(row, col))
    tp75 = sum(iou_mat[r, c] >= 0.75 for r, c in zip(row, col))

    f1_50  = 2 * tp50  / (2 * tp50 + len(pred_boxes)  - tp50  + len(gt_boxes)  - tp50)
    f1_75  = 2 * tp75  / (2 * tp75 + len(pred_boxes)  - tp75  + len(gt_boxes)  - tp75)
    mask_la = bbox_to_mask(pred_boxes, gt_mask.shape)  # ellipsoid fill
    msa, _  = mean_segmentation_accuracy(mask_la, gt_mask)
    return {"f1_50": f1_50, "f1_75": f1_75, "msa": msa}

8. Future Discussions: Fine-tuning / LoRA

Once the zero-shot baseline is established, the adaptation phase closes the domain gap

Multi-Stage Adaptation Roadmap

From Boxes to Masks

Use LocateAnything boxes as prompts to SAM decoder (or Cellpose) to produce instance masks — combine LA's zero-shot knowledge with domain-tuned segmentation head.

Multi-Modal Prompts

Extend prompts with domain context: "In electron microscopy, mitochondria are...". Use chain-of-thought in the prompt to guide the VLM toward biological structure priors.

Context-Aware Neighbor Slices

Feed 3 adjacent slices to the VLM simultaneously. MoonViT's resolution flexibility means 3-slice context patches could reduce inter-slice inconsistency during 3D wrapping.

Synthetic Training Data Generation

Use LA outputs (with human correction) as seed annotations to build a larger EM training set. Active learning loop: predict → correct → retrain → re-predict.
