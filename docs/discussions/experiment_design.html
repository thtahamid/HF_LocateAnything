Baseline Experiment Design

Controlled evaluation of LocateAnything (NVIDIA) on electron microscopy segmentation benchmarks — Lucchi++ & Kasthuri++ — with Micro-SAM as domain-specific baseline

LocateAnything-3B Micro-SAM Baseline EM Microscopy
01 Goal & Objective

Establish a rigorous experimental framework to evaluate LocateAnything's zero-shot generalization to electron microscopy segmentation, identify performance gaps against domain-adapted baselines, and chart a path toward microscopy-specific fine-tuning.

Research Question

How well does a general-purpose vision-language grounding model (trained on natural images) perform on domain-specific electron microscopy segmentation tasks without any domain adaptation?

Hypothesis

LocateAnything will achieve moderate detection recall on microscopy images due to domain shift, but will significantly underperform Micro-SAM which was fine-tuned on bio-EM data. Natural language prompting flexibility will partially compensate for accuracy gaps.

Key Insight

LocateAnything outputs boxes & points, not segmentation masks. The critical challenge is bridging the gap between detection-level outputs and pixel-level ground truth annotations.

02 Model Architecture

LocateAnything uses a novel Vision-Language architecture with Parallel Box Decoding (PBD) for high-throughput visual grounding. Understanding its structure is critical to interpreting experiment results.

Parallel Box Decoding (PBD)

Unlike standard autoregressive models that emit one coordinate at a time, PBD decodes all 4 bounding-box coordinates as a single 6-token atomic block — achieving 12.7 boxes/sec on H100 (2.5× faster than AR).

⚠️ Architecture-Segmentation Mismatch

LocateAnything is a detection/grounding model. It outputs bounding boxes and points — not pixel-level masks. For EM segmentation evaluation, boxes must be compared against mask-derived boxes, or fed as prompts to a mask generator (Micro-SAM).

03 Benchmark Datasets

Two standard electron microscopy benchmarks for evaluating subcellular/organelle segmentation in connectomics research.

Lucchi++ — Properties

  • 1024 × 768 px, 8-bit grayscale
  • Binary masks: 0 (background) / 255 (mitochondria)
  • 165 train + 165 test = 330 images
  • Each image contains multiple mitochondria instances
  • Serial-section FIB-SEM of neural tissue

Kasthuri++ — Properties

  • 1334 × 1553 px, 8-bit grayscale
  • Labeled masks: 0 (bg) / 2 (boundaries) / 255 (cells)
  • 85 train + 75 test = 160 images
  • Larger field-of-view, more complex structures
  • Volume EM of mouse/rat brain cortex
04 Experiment Matrix

Four controlled experiments designed to isolate different dimensions of performance: detection accuracy, box-to-mask conversion, resolution sensitivity, and vocabulary robustness.

Experiment 1 — Detection-as-Segmentation Baseline (click to expand)

Objective: Measure LocateAnything's zero-shot detection accuracy on EM images by converting the problem from mask prediction to box prediction.


Procedure:

  • Extract ground-truth bounding boxes from segmentation masks (one box per connected component)
  • Run LocateAnything with multiple prompt strategies across both datasets
  • Match predicted boxes to GT boxes using Hungarian matching at IoU@{0.5, 0.75}
  • Compute detection-level precision, recall, F1

Prompts to test:

# Lucchi++ prompts (mitochondria) "mitochondria" "detect all mitochondria in this electron microscopy image" "mitochondrion, organelle, membrane-bound structure" # Kasthuri++ prompts (cells/synapses) "cell" "cell membrane boundary" "synapse" "neuron"
Experiment 2 — Point-to-Mask Pipeline (click to expand)

Objective: Evaluate a two-stage pipeline where LocateAnything generates points that prompt Micro-SAM for mask generation.


Procedure:

  • Run LocateAnything in point-predicting mode to get center points of detected structures
  • Feed predicted points as prompts to Micro-SAM
  • Compare masks generated from: (a) LocateAnything points, (b) GT centroids [upper bound], (c) random GT-inside points [lower bound]
  • Measure end-to-end latency and mask quality
Experiment 3 — Resolution Sensitivity (click to expand)

Objective: Quantify how LocateAnything's performance varies with input image resolution — critical for EM images with unusual aspect ratios.


Resolution levels:

  • Native: 1024×768 (Lucchi) / 1334×1553 (Kasthuri)
  • 1000px: Short-side resized to 1000px (LocateAnything default)
  • 512px: Short-side resized to 512px (typical SAM input)
  • 2000px: Upscaled to test super-native handling

Measurement: IoU and detection recall at each resolution, holding all other parameters constant.

Experiment 4 — Vocabulary Robustness (click to expand)

Objective: Test how domain-specific biological terminology affects detection — determines whether users need prompt engineering expertise.

05 Technical Pipeline

Implementation blueprint for the evaluation infrastructure — from data loading through metric computation and visualization.

#!/usr/bin/env python3 # experiments/run_baseline.py import numpy as np from PIL import Image from transformers import AutoTokenizer, AutoProcessor, AutoModel import torch class BaselineExperiment: """Run LocateAnything on EM segmentation benchmarks.""" def __init__(self, model_id="nvidia/LocateAnything-3B"): self.tokenizer = AutoTokenizer.from_pretrained(model_id, trust_remote_code=True) self.processor = AutoProcessor.from_pretrained(model_id, trust_remote_code=True) self.model = AutoModel.from_pretrained( model_id, torch_dtype=torch.bfloat16, trust_remote_code=True ).cuda().eval() def run_inference(self, image_path, prompt, mode="hybrid"): # Load and preprocess EM image img = Image.open(image_path).convert("RGB") inputs = self.processor(images=img, text=prompt, return_tensors="pt").cuda() # Generate with specified inference mode with torch.no_grad(): output = self.model.generate(**inputs, inference_mode=mode) # Parse <box> and <point> tokens return self.parse_output(output) def run_dataset(self, dataset_dir, prompts, mode="hybrid"): # Iterate all test images, run all prompts results = {} for img_path in sorted(Path(dataset_dir).glob("*.png")): for prompt in prompts: boxes, points = self.run_inference(img_path, prompt, mode) results[(img_path.name, prompt)] = {"boxes": boxes, "points": points} return results

Step 1: Data Prep

  • Load Test_In images (grayscale → RGB)
  • Load Test_Out masks
  • Extract GT boxes from masks (connected components)
  • Compute GT centroids for point prompts

Step 2: Inference

  • Load model (BF16, CUDA)
  • For each image × prompt × mode combo
  • Capture boxes, points, latency, VRAM
  • Store raw outputs as JSON

Step 3: Evaluate

  • Match predictions to GT (Hungarian)
  • Compute IoU, Dice, ASSD, HD95
  • Aggregate per-image → per-dataset
  • Generate plots + tables

09 Future Directions & Roadmap

Beyond baseline evaluation — the path from zero-shot inference to microscopy-optimized segmentation through fine-tuning and architectural adaptation.

Phase 1 — Baseline (Current)

Run all 4 experiments, establish zero-shot performance metrics, identify failure modes. Deliver: full results report in /docs.

Phase 2 — Prompt Optimization

Based on Exp 4 results, develop microscopy-specific prompt templates. Evaluate on held-out test sets. Goal: +5-10% recall via prompting alone.

Phase 3 — LoRA Fine-tuning

Apply LoRA (rank 64, alpha 16) on Qwen2.5-3B decoder using Train_In images. Train with mask-derived boxes as ground truth. Expected: +15-25% IoU improvement.

Phase 4 — Box-to-Mask Head

Attach lightweight mask decoder (similar to SAM's prompt encoder path) on top of LocateAnything's bounding box output. End-to-end trainable.

Phase 5 — Full Domain Adaptation

Full fine-tuning with expanded microscopy datasets (beyond Lucchi++/Kasthuri++). Publish results as microscopy-specific checkpoint.
